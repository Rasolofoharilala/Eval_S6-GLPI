# Résumé du projet GLPI API + Vue TypeScript

## 1. Objectif du projet

Le but du projet est de connecter une application Vue TypeScript à l'API REST de GLPI 11.0.7.

L'application doit pouvoir :

* communiquer avec GLPI via OAuth2 ;
* générer automatiquement un token ;
* appeler les endpoints de GLPI ;
* récupérer des données comme les utilisateurs, tickets et ordinateurs ;
* structurer le code proprement avec plusieurs couches ;
* automatiser une partie du code avec des scripts Python.

---

```text

```

L'installation finale a été réalisée via la commande CLI GLPI :

```bash

  --db-host=localhost \
  --db-name=glpi \
  --db-user=glpi \
  --db-password=glpi \
  --no-interaction
```

## 6. Compréhension du fonctionnement OAuth2

Le client OAuth ne représente pas un utilisateur.

Il représente l'application :

```text
NewApp = application Vue / script / client API
```

Le `Client ID` identifie l'application.

Le `Client Secret` sert de secret de sécurité pour cette application.

Le `Grant Type` indique la méthode utilisée pour demander un token.

Le `Scope` indique les permissions accordées au token.

L'`Access Token` est le jeton temporaire utilisé pour appeler l'API.

Le `Refresh Token` permet de demander un nouvel access token sans refaire toute l'authentification.

Flux simplifié :

```text
Client ID + Client Secret + Username + Password + Scopes
↓
GLPI vérifie les informations
↓
GLPI retourne un access_token
↓
Vue utilise ce token dans le header Authorization
```

Exemple de header :

```http
Authorization: Bearer ACCESS_TOKEN
```

## 7. Test de l'API avec curl

Un token OAuth a été généré avec :

```bash
curl -X POST http://localhost/api.php/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password" \
  -d "client_id=CLIENT_ID" \
  -d "client_secret=CLIENT_SECRET" \
  -d "username=glpi" \
  -d "password=glpi" \
  -d "scope=api user email status graphql"
```

Test de l'utilisateur connecté :

```bash
curl http://localhost/api.php/v2.3/Administration/User/Me \
  -H "Authorization: Bearer $TOKEN"
```

Test des tickets :

```bash
curl http://localhost/api.php/v2.3/Assistance/Ticket \
  -H "Authorization: Bearer $TOKEN"
```

Résultats validés :

```text
/Administration/User/Me    OK
/Administration/User       OK
/Assistance/Ticket         OK
/Assets/Computer           OK
```

## 9. Tentative avec openapi-typescript

Objectif :

```text
Générer automatiquement les types TypeScript depuis swagger.json
```

Commande utilisée :

```bash
npx --yes openapi-typescript docs/swagger.json \
  -o src/generated/glpi-api-types.ts
```

Problème rencontré :

```text
Can't resolve $ref
```

Cause :

```text
Le Swagger GLPI contient certaines références cassées ou incomplètes.
```

Conclusion :

```text
openapi-typescript n'est pas fiable directement avec ce Swagger GLPI.
```

Solution retenue :

```text
Utiliser des scripts Python personnalisés.
```

## 12. Couche api/

La couche `api/` contient la logique technique de communication avec GLPI.

### glpiConfig.ts

Ce fichier lit les variables d'environnement.

```ts
export const glpiConfig = {
  apiUrl: import.meta.env.VITE_GLPI_API_URL,
  tokenUrl: import.meta.env.VITE_GLPI_TOKEN_URL,

  clientId: import.meta.env.VITE_GLPI_CLIENT_ID,
  clientSecret: import.meta.env.VITE_GLPI_CLIENT_SECRET,

  username: import.meta.env.VITE_GLPI_USERNAME,
  password: import.meta.env.VITE_GLPI_PASSWORD,

  scope: import.meta.env.VITE_GLPI_SCOPE,
}
```

### tokenManager.ts

Ce fichier gère la génération du token OAuth.

Il est responsable de :

```text
- demander un access_token ;
- stocker temporairement le token ;
- vérifier si le token est expiré ;
- redemander un token si nécessaire.
```

### httpClient.ts

Ce fichier configure Axios.

Il ajoute automatiquement :

```http
Authorization: Bearer TOKEN
```

à chaque requête.

### crudClient.ts

Ce fichier contient les fonctions CRUD génériques :

```ts
getAll()
getById()
create()
update()
remove()
```

## 19. Où mettre la logique métier ?

La logique métier ne doit pas être placée directement dans les composants Vue.

Elle doit être placée dans :

```text
src/domain/
```

Exemple :

```text
src/domain/ticket/ticketMapper.ts
src/domain/ticket/ticketRules.ts
src/domain/user/userMapper.ts
src/domain/computer/computerMapper.ts
```

Exemple :

```ts
export function mapTicketForDisplay(ticket: any) {
  return {
    id: ticket.id,
    title: ticket.name || 'Ticket sans titre',
    status: ticket.status?.name ?? 'Inconnu',
    priority: ticket.priority,
    requester: ticket.user_recipient?.name ?? 'Non défini',
    createdAt: ticket.date_creation,
  }
}
```
